# 知识图谱 - 核心概念提取

> **项目定位**：知识梳理与理论构建项目（非编程项目）
> **最后更新**：2025-02-02
> **用途**：提取项目中的核心概念和关系，为知识图谱构建提供基础数据

## 一、核心概念列表

### 1.1 Modern目录核心概念

#### MSMFIT相关概念

- **MSMFIT**（最小语义模型）
  - 类型：Model
  - 定义：`MSMFIT = {E, R, V, C}`
  - 文档：[Modern/01-IT语义世界基础理论/02-最小语义模型MSMFIT.md](Modern/01-IT语义世界基础理论/02-最小语义模型MSMFIT.md)

- **E（Entity）**（语义实体）
  - 类型：Component
  - 定义：不可再分的业务概念原子
  - 关系：isPartOf(MSMFIT)

- **R（Relation）**（语义关系）
  - 类型：Component
  - 定义：实体间的稳定关联
  - 关系：isPartOf(MSMFIT), connects(E, E)

- **V（Event）**（语义事件）
  - 类型：Component
  - 定义：带有时间戳的状态变迁
  - 关系：isPartOf(MSMFIT), triggers(R)

- **C（Context）**（语义上下文）
  - 类型：Component
  - 定义：修饰语义解释的环境参数
  - 关系：isPartOf(MSMFIT), modifies(E), modifies(R)

#### SMDD相关概念

- **SMDD**（语义模型驱动设计）
  - 类型：Methodology
  - 定义：通过构建语义模型驱动系统设计的方法
  - 文档：[Modern/02-语义驱动架构理论/01-语义模型驱动设计SMDD.md](Modern/02-语义驱动架构理论/01-语义模型驱动设计SMDD.md)

- **语义保真原则**
  - 类型：Principle
  - 定义：模型语义必须精确映射问题域的核心需求
  - 关系：isPrincipleOf(SMDD)

- **可组合性原则**
  - 类型：Principle
  - 定义：语义模型必须支持原子语义的复合与编排
  - 关系：isPrincipleOf(SMDD)

#### DIKWP相关概念

- **DIKWP模型**
  - 类型：Model
  - 定义：数据→信息→知识→智慧→目的的五层模型
  - 文档：[Modern/02-语义驱动架构理论/04-DIKWP语义驱动编程范式.md](Modern/02-语义驱动架构理论/04-DIKWP语义驱动编程范式.md)

- **D（Data）**（数据）
  - 类型：Layer
  - 定义：原始业务事实
  - 关系：isLayerOf(DIKWP), transformsTo(I)

- **I（Information）**（信息）
  - 类型：Layer
  - 定义：带语义的实体与关系
  - 关系：isLayerOf(DIKWP), transformsTo(K)

- **K（Knowledge）**（知识）
  - 类型：Layer
  - 定义：业务规则与约束
  - 关系：isLayerOf(DIKWP), transformsTo(W)

- **W（Wisdom）**（智慧）
  - 类型：Layer
  - 定义：基于语境的决策逻辑
  - 关系：isLayerOf(DIKWP), transformsTo(P)

- **P（Purpose）**（目的）
  - 类型：Layer
  - 定义：业务目标与意图驱动
  - 关系：isLayerOf(DIKWP)

#### 可逆计算相关概念

- **可逆计算**
  - 类型：Paradigm
  - 定义：DSL ⇌ Code的双向转换
  - 文档：[Modern/03-业务语义与技术实现同构理论/02-桥接机制与DSL转换器.md](Modern/03-业务语义与技术实现同构理论/02-桥接机制与DSL转换器.md)

- **DSL**（领域特定语言）
  - 类型：Language
  - 定义：用于表达业务语义的专用语言
  - 关系：transformsTo(Code), transformsFrom(Code)

- **正向转换**
  - 类型：Transformation
  - 定义：DSL → 代码生成
  - 关系：isTransformationOf(可逆计算)

- **反向转换**
  - 类型：Transformation
  - 定义：代码 → DSL提取
  - 关系：isTransformationOf(可逆计算)

- **差量更新**
  - 类型：Mechanism
  - 定义：增量代码生成
  - 关系：isMechanismOf(可逆计算)

### 1.2 Analysis目录核心概念

#### 形式化方法相关概念

- **形式化方法**
  - 类型：Methodology
  - 定义：使用数学方法描述和验证系统的方法
  - 文档：[Analysis/06-形式模型理论体系/00-形式模型理论体系总论.md](Analysis/06-形式模型理论体系/00-形式模型理论体系总论.md)

- **形式化证明**
  - 类型：Method
  - 定义：使用逻辑推理证明系统正确性的方法
  - 关系：isMethodOf(形式化方法)

- **模型检查**
  - 类型：Method
  - 定义：自动验证系统模型是否满足性质的方法
  - 关系：isMethodOf(形式化方法)

- **定理证明**
  - 类型：Method
  - 定义：使用逻辑推理证明定理的方法
  - 关系：isMethodOf(形式化方法)

#### 软件架构相关概念

- **软件架构**
  - 类型：Concept
  - 定义：软件系统的结构和组织方式
  - 文档：[Analysis/06-软件架构理论体系/00-软件架构理论体系总论.md](Analysis/06-软件架构理论体系/00-软件架构理论体系总论.md)

- **组件**
  - 类型：Element
  - 定义：软件系统的基本构建块
  - 关系：isElementOf(软件架构)

- **连接器**
  - 类型：Element
  - 定义：组件间的连接机制
  - 关系：isElementOf(软件架构), connects(组件, 组件)

- **架构模式**
  - 类型：Pattern
  - 定义：可重用的架构设计方案
  - 关系：isPatternOf(软件架构)

```

  ├── hasComponent(E)

  ├── isModifiedBy(C)

  ├── hasSubject(E)
  ├── hasObject(E)
  └── hasContext(C)

C
  ├── modifies(E)
  ├── modifies(R)
  └── modifies(V)
```

## 三、RDF/OWL格式定义（草案）

### 3.1 命名空间定义

```turtle
@prefix : <http://formal-architecture.org/ontology#> .
@prefix rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
@prefix owl: <http://www.w3.org/2002/07/owl#> .
@prefix xsd: <http://www.w3.org/2001/XMLSchema#> .
```

### 3.2 核心类定义

```turtle
# MSMFIT类
:MSMFIT rdf:type owl:Class ;
        rdfs:label "最小语义模型"@zh ;
        rdfs:comment "MSMFIT = {E, R, V, C}"@zh .

:Entity rdf:type owl:Class ;
        rdfs:label "语义实体"@zh ;
        rdfs:subClassOf :MSMFITComponent .

:Relation rdf:type owl:Class ;
          rdfs:label "语义关系"@zh ;
          rdfs:subClassOf :MSMFITComponent .

:Event rdf:type owl:Class ;
       rdfs:label "语义事件"@zh ;
       rdfs:subClassOf :MSMFITComponent .

:Context rdf:type owl:Class ;
          rdfs:label "语义上下文"@zh ;
          rdfs:subClassOf :MSMFITComponent .

# SMDD类
:SMDD rdf:type owl:Class ;
      rdfs:label "语义模型驱动设计"@zh .

:SemanticFidelityPrinciple rdf:type owl:Class ;
                           rdfs:label "语义保真原则"@zh ;
                           rdfs:subClassOf :Principle .

:ComposabilityPrinciple rdf:type owl:Class ;
                        rdfs:label "可组合性原则"@zh ;
                        rdfs:subClassOf :Principle .
```

### 3.3 属性定义

```turtle
# MSMFIT属性
:hasComponent rdf:type owl:ObjectProperty ;
              rdfs:domain :MSMFIT ;
              rdfs:range :MSMFITComponent .

:connects rdf:type owl:ObjectProperty ;
          rdfs:domain :Relation ;
          rdfs:range :Entity .

:triggers rdf:type owl:ObjectProperty ;
          rdfs:domain :Relation ;
          rdfs:range :Event .

:modifies rdf:type owl:ObjectProperty ;
          rdfs:domain :Context ;
          rdfs:range :MSMFITComponent .

# SMDD属性
:uses rdf:type owl:ObjectProperty ;
      rdfs:domain :SMDD ;
      rdfs:range :MSMFIT .

:follows rdf:type owl:ObjectProperty ;
         rdfs:domain :SMDD ;
         rdfs:range :Principle .
```

### 3.4 实例示例

```turtle
# MSMFIT实例
:msmfit-example rdf:type :MSMFIT ;
                :hasComponent :entity-user ;
                :hasComponent :relation-purchases ;
                :hasComponent :event-payment ;
                :hasComponent :context-vip .

:entity-user rdf:type :Entity ;
             rdfs:label "用户实体"@zh .

:relation-purchases rdf:type :Relation ;
                    rdfs:label "购买关系"@zh ;
                    :connects :entity-user, :entity-product .

# SMDD实例
:smdd-example rdf:type :SMDD ;
              :uses :msmfit-example ;
              :follows :semantic-fidelity-principle ;
              :follows :composability-principle .
```
