## 一、知识体系思维导图

```text
数据存储与数据库架构
│
├─► 关系型数据库 (RDBMS)
│   ├─► 存储引擎: B+树索引、LSM-Tree、堆文件
│   ├─► 事务处理: ACID、2PL、MVCC、ARIES恢复
│   ├─► 查询优化: 代价模型、Join算法、计划空间
│   └─► 代表: PostgreSQL, MySQL/InnoDB, Oracle, SQL Server
│
├─► NoSQL 数据库
│   ├─► 键值存储: Redis, RocksDB, DynamoDB
│   ├─► 文档数据库: MongoDB, Couchbase, Firestore
│   ├─► 列族存储: Cassandra, HBase, Bigtable
│   └─► 图数据库: Neo4j, TigerGraph, Dgraph
│
├─► NewSQL / 分布式数据库
│   ├─► Google Spanner: TrueTime + 外部一致性
│   ├─► CockroachDB: 全局排序的分布式KV
│   ├─► TiDB: Raft + MVCC + 分布式SQL
│   └─► YugabyteDB: DocDB存储 + Raft共识
│
├─► 数据架构模式
│   ├─► 数据网格 (Data Mesh): 领域所有权 + 自服务
│   ├─► 数据湖 (Data Lake): 原始格式存储
│   ├─► 湖仓一体 (Lakehouse): Delta Lake / Iceberg / Hudi
│   └─► 数据仓库 (Data Warehouse): 星型/雪花模型
│
└─► 分布式事务
    ├─► 2PC (两阶段提交): 阻塞问题
    ├─► 3PC (三阶段提交): 超时解阻塞
    ├─► TCC (Try-Confirm-Cancel): 业务补偿
    ├─► Saga: 长事务的补偿模式
    └─► Percolator: 基于Bigtable的分布式事务
```

## 二、数据库系统的形式化分层

```text
数据库系统 DBMS = ⟨Storage, Index, Query, Transaction, Recovery⟩

存储层 (Storage):
  页式管理: 磁盘 ↔ Buffer Pool ↔ 页表
  替换策略: LRU, CLOCK, 2Q

索引层 (Index):
  B+树: 磁盘友好、范围查询、有序
  LSM-Tree: 写优化、分层合并、读放大
  哈希索引: O(1)点查、无范围查询

查询层 (Query):
  解析 → 逻辑计划 → 优化器 → 物理计划 → 执行器
  代价模型: C = I/O_cost + CPU_cost + Network_cost

事务层 (Transaction):
  ACID: 原子性、一致性、隔离性、持久性
  隔离级别: 读未提交 → 读已提交 → 可重复读 → 串行化

恢复层 (Recovery):
  WAL (Write-Ahead Logging)
  ARIES: Analysis → Redo → Undo
```

## 六、批判性总结

数据库架构的演进本质是**一致性-可用性-分区容错**的永恒权衡。RDBMS 追求 ACID，NoSQL 追求 BASE，NewSQL 试图在分布式环境下恢复 ACID——Spanner 用 TrueTime（GPS+原子钟）"购买"了一致性，证明**时钟同步硬件可以购买一致性**。

数据网格 (Data Mesh) 是 DDD 思想在数据架构中的延伸：将数据所有权从中心化平台团队下放到领域团队。这不是技术问题，而是**组织问题**——康威定律在数据领域的再次验证。
