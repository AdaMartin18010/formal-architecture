## 一、知识体系思维导图

```text
设计系统的代数结构
│
├─► 设计令牌 (Design Tokens) 的代数结构
│   ├─ 定义: 𝓓 = ⟨T, ⊕, ⊗, 𝟎, 𝟏⟩
│   │   T: 设计令牌集合 (colors, spacing, typography)
│   │   ⊕: 组合操作 (color + opacity)
│   │   ⊗: 应用操作 (token → component)
│   │   𝟎: 中性元素 (透明, 零间距)
│   │   𝟏: 单位元素 (默认主题)
│   │
│   └─► 定理 T7: ⟨T, ⊕⟩ 构成阿贝尔群
│       ├─ 封闭性: ∀a,b ∈ T, a ⊕ b ∈ T
│       ├─ 结合律: (a ⊕ b) ⊕ c = a ⊕ (b ⊕ c)
│       ├─ 交换律: a ⊕ b = b ⊕ a
│       ├─ 单位元: ∃𝟎 ∈ T: a ⊕ 𝟎 = a
│       └─ 逆元: ∀a ∈ T, ∃a⁻¹ ∈ T: a ⊕ a⁻¹ = 𝟎
│
├─► 组件组合的幺半群 (Monoid) 结构
│   ├─ 定义: ⟨C, ∘, ε⟩
│   │   C: 组件集合
│   │   ∘: 组合操作 (嵌套/包裹)
│   │   ε: 空组件/Fragment
│   │
│   ├─ 结合律: (A ∘ B) ∘ C = A ∘ (B ∘ C)
│   ├─ 单位元: A ∘ ε = ε ∘ A = A
│   └─ ❌ 交换律不成立: A ∘ B ≠ B ∘ A
│
├─► 高阶组件 (HOC) 是群上的自同态
│   └─ HOC: C → C, withTheme(A) = ThemeProvider ∘ A ∘ ThemeConsumer
│
└─► CSS 变量 (Custom Properties) 是群的线性表示
    ├─ --color-primary: #000;
    ├─ --spacing-unit: 4px;
    └─ 这些变量是群 T 的生成元 (Generators)
```

## 二、设计令牌的群论证明

### 定理 T7: 设计令牌在 ⊕ 操作下构成阿贝尔群

```text
证明:

封闭性: ∀a,b ∈ T, a ⊕ b ∈ T
  例: color(#000) ⊕ opacity(0.5) = color(rgba(0,0,0,0.5)) ∈ T

结合律: (a ⊕ b) ⊕ c = a ⊕ (b ⊕ c)
  例: (spacing(4) ⊕ spacing(8)) ⊕ spacing(16) = spacing(28)
     spacing(4) ⊕ (spacing(8) ⊕ spacing(16)) = spacing(28)

交换律: a ⊕ b = b ⊕ a
  例: color ⊕ opacity = opacity ⊕ color (结果相同)

单位元: ∃𝟎 ∈ T: a ⊕ 𝟎 = a
  𝟎 = transparent, spacing(0), font-size(inherit)

逆元: ∀a ∈ T, ∃a⁻¹ ∈ T: a ⊕ a⁻¹ = 𝟎
  例: spacing(8) 的逆元是 spacing(-8) (在相对定位中)
     color 的逆元是 mix-blend-mode: difference 中的补色

∴ ⟨T, ⊕⟩ 是阿贝尔群    ∎
```

## 三、组件组合的幺半群

```text
定义: 组件组合 ⟨C, ∘, ε⟩
  C: 组件集合
  ∘: 组合操作 (嵌套/包裹)
  ε: 空组件/Fragment

公理验证:
  结合律: (A ∘ B) ∘ C = A ∘ (B ∘ C)
    <A><B><C /></B></A> = <A><B><C /></B></A>

  单位元: A ∘ ε = ε ∘ A = A
    <A><></></A> = <A /> = <></><A /></>

但交换律不成立:
  A ∘ B ≠ B ∘ A (组件嵌套顺序影响渲染树结构)

∴ ⟨C, ∘, ε⟩ 是幺半群, 非阿贝尔群

高阶组件 (HOC) 是群上的自同态:
  HOC: C → C
  withTheme(A) = ThemeProvider ∘ A ∘ ThemeConsumer
  即: HOC 是组件群上的群同态 (保持组合结构)
```

## 六、批判性总结

设计系统的代数结构揭示了 UI 设计的深层数学规律。Design Tokens 的阿贝尔群性质保证了样式组合的可预测性；组件的幺半群性质解释了为什么组件嵌套顺序不可交换。将这些抽象结构显式化，有助于构建可数学验证的设计系统工具链。
