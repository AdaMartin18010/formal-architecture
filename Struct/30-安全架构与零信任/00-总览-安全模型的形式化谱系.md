## 二、零信任架构的形式化

```text
定义 (零信任访问决策):
  AccessDecision = f(Identity, Device, Context, Resource, Policy)

  Identity: 用户/服务身份 (已认证)
  Device: 设备状态 (合规性、健康度)
  Context: 上下文 (时间、地点、网络、行为异常)
  Resource: 请求的资源敏感性
  Policy: 动态策略 (ReBAC/ABAC)

  决策输出: {Allow, Deny, StepUpAuth, LimitAccess}

  与传统边界模型的区别:
    传统: InsideNetwork → Trust → AllowAll
    零信任: ∀Request → Verify → PolicyDecision
```

## 六、批判性总结

零信任不是产品，而是**安全范式的根本转变**：从"信任但验证"到"永不信任，始终验证"。但零信任的实施面临**组织阻力**——它要求所有访问都经过认证和授权，这在传统企业中意味着巨大的流程重构。

JWT 的广泛滥用是 2026 年最大的应用安全隐患之一：开发者将 JWT 用作会话存储（放入大量声明），却忽视了**JWT 无法撤销**的本质。一旦泄露，攻击者在令牌过期前拥有完全权限。这印证了安全领域的永恒真理：**便利性 vs 安全性**的权衡中，工程师往往过度选择前者。

## 八、形式化推理链

### 8.1 公理 → 引理 → 定理 → 推论

**公理A1** (Saltzer & Schroeder, 1975 — 经济机制原则):
> 安全机制的设计应尽可能简单。每增加一个组件，就增加一个被攻击面。

**公理A2** (Shannon, 1949 — 完美保密):
> 若密钥长度 ≥ 消息长度且仅用一次，则系统达到信息论安全：H(M|C) = H(M)。

**引理L1** (BLP基本安全定理, Bell & LaPadula, 1976):
> 若系统初始状态安全，且所有状态转换规则均保持简单安全属性(Simple Security Property)和*-属性(*-Property)，则系统所有可达状态均安全。

形式化: ∀s₀ ∈ SecureState, ∀ rule ∈ Rules:
       secure(s₀) ∧ preserve(rule, secure) → ∀s ∈ Reachable(s₀): secure(s)

**引理L2** (Biba对偶性, Biba, 1977):
> 将BLP的"no read up, no write down"规则取逆（镜像），即可得到保护完整性的"no read down, no write up"。

**定理T1** (访问控制模型包含关系):
> RBAC ⊂ ABAC ⊂ ReBAC（表达能力层面）。
> 证明: RBAC的Role可表示为ABAC的单一属性；ABAC的策略可编码为ReBAC的关系约束。

**定理T2** (前向保密的组合安全性, Krawczyk, 2010):
> 在CDH假设下，(EC)DHE密钥交换 + HKDF密钥派生 → 会话密钥满足前向保密。
> 即: 长期私钥泄露 ⟹̸ 历史会话密钥可计算。

**推论C1**:
> TLS 1.3 强制 (EC)DHE，故在所有合规实现中，历史流量 confidentiality 在私钥泄露后仍保持。

**推论C2**:
> 零信任的"永不信任，始终验证"原则，本质上是将BLP的MAC策略从"位置属性"迁移到"多维度上下文属性"。

## 十一、深度批判性总结

安全模型的形式化谱系揭示了计算机安全领域一个深刻的认知演进：从**静态、二元的访问控制**走向**动态、连续的信任评估**。Bell-LaPadula (1973-1976) 和 Biba (1977) 作为状态机模型的双生子，分别用数学精确性定义了机密性与完整性，但它们的刚性也暴露无遗——BLP的"no read up, no write down"在商用环境中导致信息共享困难，Biba的"no read down, no write up"阻碍了高完整性进程读取外部输入。这两种模型的共同局限在于：它们假设安全等级是静态分配的，忽略了现代计算中上下文持续变化的本质。

Harrison-Ruzzo-Ullman (1976) 和 Take-Grant (1977) 将形式化从状态空间推向命令序列与权限流，但这些工作仍停留在"操作系统内核安全"的狭隘范畴。真正打破框架的是Diffie-Hellman (1976) 的公钥密码学思想——它证明了**安全通信不必依赖预先共享的秘密**，这一洞见直接催生了TLS、VPN和整个互联网商业生态。然而，密码学的数学保证与系统安全的工程现实之间存在永恒的鸿沟：Heartbleed、Shellshock、Log4j等灾难性漏洞表明，最精致的密码协议也可能因实现层面的缓冲区溢出或注入攻击而土崩瓦解。

零信任架构 (Kindervag, 2010; NIST SP 800-207, 2020) 代表了安全哲学从"防御边界"到"防御每个事务"的范式跃迁。其形式化内核——将访问决策函数从单一网络位置扩展到多维上下文向量——在理论上优雅，但在实践中面临严峻的**可用性税**和**可组合性挑战**。当每个微服务调用都需要策略评估时，延迟累积和策略冲突成为系统架构的瓶颈。2026年的前沿探索是将形式化验证（如TLA+对Zanzibar策略的模型检验）与机器学习驱动的异常检测相结合，但这又引入了新的可信计算基础问题：谁验证验证者？安全架构的终极悖论或许在于——**任何形式化系统都无法形式化地证明自身的完备性**，这正是Gödel不完备定理在安全工程中的幽灵回响。
