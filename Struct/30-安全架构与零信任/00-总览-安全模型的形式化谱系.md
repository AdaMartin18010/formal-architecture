## 一、知识体系思维导图

```text
安全架构与零信任
│
├─► 身份与访问管理 (IAM)
│   ├─► 认证 (Authentication)
│   │   ├─ 密码: bcrypt/Argon2/scrypt (慢哈希)
│   │   ├─ 多因素认证 (MFA): TOTP/HOTP/WebAuthn/FIDO2
│   │   ├─ SSO: SAML / OAuth 2.0 / OIDC
│   │   └─ 无密码: Passkeys (WebAuthn/生物识别)
│   │
│   ├─► 授权 (Authorization)
│   │   ├─ RBAC: 基于角色的访问控制
│   │   ├─ ABAC: 基于属性的访问控制
│   │   ├─ ReBAC: 基于关系的访问控制 (Google Zanzibar)
│   │   └─ Policy-as-Code: OPA / Cedar
│   │
│   └─► JWT (JSON Web Token)
│       ├─ 结构: Header.Payload.Signature
│       ├─ 安全: HS256 (对称) / RS256/ES256 (非对称)
│       └─ 风险: 泄露即授权、无法撤销、Payload过大
│
├─► 传输安全
│   ├─► TLS 1.3
│   │   ├─ 握手: 1-RTT (首次) / 0-RTT (恢复)
│   │   ├─ 完美前向保密 (PFS): ECDHE
│   │   └─ 证书: X.509、Let's Encrypt、证书透明度
│   │
│   └─► mTLS (双向 TLS)
│       ├─ 服务间认证: 证书即身份
│       └─ 适用: 服务网格、微服务内网通信
│
├─► 应用安全
│   ├─► 常见漏洞 (OWASP Top 10)
│   │   ├─ 注入: SQL/NoSQL/OS Command Injection
│   │   ├─ 失效访问控制: IDOR、水平/垂直越权
│   │   ├─ 敏感数据暴露: 加密缺失、明文传输
│   │   ├─ XSS: 存储型/反射型/DOM型
│   │   └─ CSRF: 跨站请求伪造
│   │
│   └─► 安全开发生命周期 (SSDLC)
│       ├─ SAST: 静态应用安全测试
│       ├─ DAST: 动态应用安全测试
│       ├─ SCA: 软件成分分析 (依赖漏洞)
│       └─ 渗透测试: 红队/蓝队
│
└─► 零信任架构 (Zero Trust)
    ├─ 核心原则: "从不信任，始终验证" (Never Trust, Always Verify)
    ├─ 假设 breach: 边界已失守、内部不可信
    ├─ 最小权限: 按需授权、动态评估
    ├─ 持续验证: 行为分析、风险评估
    └─ 代表: BeyondCorp (Google), Azure AD, Okta
```

## 二、零信任架构的形式化

```text
定义 (零信任访问决策):
  AccessDecision = f(Identity, Device, Context, Resource, Policy)

  Identity: 用户/服务身份 (已认证)
  Device: 设备状态 (合规性、健康度)
  Context: 上下文 (时间、地点、网络、行为异常)
  Resource: 请求的资源敏感性
  Policy: 动态策略 (ReBAC/ABAC)

  决策输出: {Allow, Deny, StepUpAuth, LimitAccess}

  与传统边界模型的区别:
    传统: InsideNetwork → Trust → AllowAll
    零信任: ∀Request → Verify → PolicyDecision
```

## 三、安全机制对比矩阵

| 机制 | 保护对象 | 威胁模型 | 实现复杂度 | 性能影响 | 2026年趋势 |
|------|---------|---------|-----------|---------|-----------|
| **TLS 1.3** | 传输层 | 中间人窃听 | 低 | 极低 | 默认标配 |
| **mTLS** | 服务间通信 | 伪造服务身份 | 中 | 中 | 服务网格标配 |
| **JWT** | 应用层认证 | 会话劫持 | 低 | 低 | 广泛使用但有风险 |
| **Passkeys** | 用户认证 | 钓鱼、凭证填充 | 低 | 低 | **快速增长** |
| **OPA** | 策略执行 | 越权访问 | 中 | 低 | 云原生标配 |
| **WAF** | Web应用 | OWASP Top 10 | 中 | 中 | 边缘部署 |

## 六、批判性总结

零信任不是产品，而是**安全范式的根本转变**：从"信任但验证"到"永不信任，始终验证"。但零信任的实施面临**组织阻力**——它要求所有访问都经过认证和授权，这在传统企业中意味着巨大的流程重构。

JWT 的广泛滥用是 2026 年最大的应用安全隐患之一：开发者将 JWT 用作会话存储（放入大量声明），却忽视了**JWT 无法撤销**的本质。一旦泄露，攻击者在令牌过期前拥有完全权限。这印证了安全领域的永恒真理：**便利性 vs 安全性**的权衡中，工程师往往过度选择前者。
