## 一、核心概念定义

### 1.1 两种类型系统的形式化对比

```text
结构类型 (TypeScript 默认):
  公理: T₁ ≡ T₂ ⇔ ∀p ∈ properties(T₁), p ∈ properties(T₂)
  灵活性: ★★★★★
  安全性: ★★★☆☆

  反例:
    interface Dog { name: string; bark(): void }
    interface Person { name: string; bark(): void }
    // 结构等价, 语义荒谬
    const person: Person = new Dog() // 合法!

名义类型 (Java/C# 风格, TS 模拟):
  公理: T₁ ≡ T₂ ⇔ name(T₁) = name(T₂)
  灵活性: ★★★☆☆
  安全性: ★★★★★

  TS 模拟:
    type UserId = string & { readonly __brand: unique symbol }
    // 引入名义维度, 但增加语法噪音
```
