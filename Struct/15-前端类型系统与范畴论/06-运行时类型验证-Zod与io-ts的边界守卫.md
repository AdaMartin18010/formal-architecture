## 一、核心概念定义

### 1.1 双重验证策略

```text
编译时验证: TypeScript 类型系统
  - 优点: 零运行时开销
  - 缺点: 无法验证外部数据 (API 响应、localStorage)

运行时验证: Zod / io-ts
  - 优点: 边界数据安全
  - 缺点: 运行时开销、与类型系统重复定义

策略: 编译时 + 运行时双重验证
  - 内部逻辑: 仅编译时
  - 外部边界: 编译时 + 运行时
```
