# 模块间交叉引用网络

> **定位**: 本文件构建 Struct 知识体系 34 个模块之间的概念关联图谱，帮助读者理解模块间的逻辑依赖与知识迁移路径。

## 一、核心依赖链路

```text
知识依赖的拓扑排序（从基础到应用）
│
├─► 01 形式化计算理论根基
│   ├─→ 02 分布式系统不可能性 (图灵机→FLP→CAP)
│   ├─→ 07 形式化方法与验证 (λ演算→类型系统→定理证明)
│   ├─→ 15 前端类型系统与范畴论 (Curry-Howard)
│   └─→ 32 编程语言运行时 (计算模型→JVM/Go/V8)
│
├─► 02 分布式系统不可能性 + 03 共识算法 + 04 数据一致性
│   ├─→ 21 消息队列 (共识→Kafka/KRaft)
│   ├─→ 22 数据库 (CAP→ACID/BASE/NewSQL)
│   ├─→ 24 API网关/网格 (CAP→服务发现/负载均衡)
│   └─→ 28 大数据 (分区→MapReduce/Flink)
│
├─► 05 架构模式 + 06 DDD
│   ├─→ 13-20 前端体系 (分层架构→前端计算模型)
│   ├─→ 25 容器化/K8s (微服务→容器编排)
│   ├─→ 31 云原生 (微服务→IaaS/PaaS/FaaS)
│   └─→ 06 DDD → 22 数据网格 (领域边界→数据所有权)
│
├─► 09 安全模型 + 30 安全架构
│   ├─→ 25 容器化 (安全→Namespace/Capabilities)
│   ├─→ 30 零信任 (BAN→OAuth/TLS/零信任)
│   └─→ 24 服务网格 (mTLS→Sidecar安全通信)
│
├─► 08 性能模型
│   ├─→ 23 缓存 (排队论→缓存层次)
│   ├─→ 26 可观测性/SRE (Little定律→容量规划)
│   └─→ 32 运行时 (GC算法→延迟/吞吐权衡)
│
└─► 26 可观测性 + 27 DevOps
    ├─→ 25 容器化 (监控→Pod健康检查)
    ├─→ 31 云原生 (FinOps→成本观测)
    └─→ 27 DevOps → 05 架构模式 (发布策略→金丝雀/蓝绿)
```
