## 一、核心概念定义

### 1.1 微前端的形式化定义

```text
微前端: 将前端应用拆分为可独立开发、部署、运行的子应用

  形式化特征:
    - 每个微前端是独立的部署单元
    - 运行时通过容器应用 (Shell) 组合
    - 共享依赖通过 Module Federation 或 import-map 管理
```

### 1.2 微前端 ≈ 数据库分片的同构

```text
同构映射:
  数据库分片 (Sharding) → 微前端 / Module Federation
  状态按业务域分区 → 每个微前端管理自己的状态片
```
