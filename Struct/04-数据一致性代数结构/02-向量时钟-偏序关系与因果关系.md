# 向量时钟：偏序关系与因果关系

> **定位**：向量时钟是分布式系统的"逻辑时钟"——它不依赖物理时间的同步，而是通过事件的因果关系来确定顺序。它是实现因果一致性的核心机制。
>
> **核心命题**：物理时钟在分布式系统中不可靠（NTP漂移、闰秒、时钟回拨），向量时钟提供了一种不依赖物理时间的因果推理方法。

---

## 一、思维导图：向量时钟核心概念

```text
向量时钟（Vector Clock）
│
├─【核心思想】
│   └─ 每个节点维护一个向量，记录对其他节点事件的了解
│
├─【更新规则】
│   1. 本地事件：VC[i] += 1
│   2. 发送消息：附带当前VC
│   3. 接收消息：VC[j] = max(VC[j], received_VC[j]) for all j
│
├─【比较规则】
│   ├─ VC₁ < VC₂: ∀i, VC₁[i] ≤ VC₂[i] 且 ∃j, VC₁[j] < VC₂[j]
│   ├─ VC₁ = VC₂: ∀i, VC₁[i] = VC₂[i]
│   └─ 并发（不可比较）：其他情况
│
├─【性质】
│   ├─ 偏序关系（Partial Order）
│   ├─ 可确定因果关系（happens-before）
│   └─ 不可确定全序（并发事件）
│
└─【工程挑战】
    ├─ 空间开销：O(n) per event
    ├─ 垃圾回收：删除已确认的事件记录
    └─ 与物理时钟结合：Hybrid Logical Clocks
```

## 二、向量时钟的形式化定义

```
定义：
  设系统有N个进程 {P₁, P₂, ..., Pₙ}
  每个进程Pᵢ维护向量 VCᵢ = [c₁, c₂, ..., cₙ]

更新规则：
  1. Pᵢ发生本地事件：
     VCᵢ[i] += 1

  2. Pᵢ发送消息给Pⱼ：
     消息附带 VCᵢ

  3. Pⱼ接收来自Pᵢ的消息（附带VC_msg）：
     ∀k: VCⱼ[k] = max(VCⱼ[k], VC_msg[k])
     VCⱼ[j] += 1

偏序关系：
  VC₁ ≤ VC₂ ⟺ ∀i: VC₁[i] ≤ VC₂[i]

  VC₁ < VC₂ ⟺ VC₁ ≤ VC₂ ∧ VC₁ ≠ VC₂

  VC₁ || VC₂（并发）⟺ ¬(VC₁ ≤ VC₂) ∧ ¬(VC₂ ≤ VC₁)

因果关系：
  e₁ → e₂（e₁ happens-before e₂）⟺ VC(e₁) < VC(e₂)

  e₁ || e₂（并发）⟺ 无法确定因果关系
```

## 三、向量时钟示例

```
3节点系统（A, B, C）的向量时钟演进：

  A: [1,0,0] ──m1──► B: [1,1,0]
                        │
                        ▼
  A: [2,0,0] ◄──m2── C: [1,1,1]
  │                     ▲
  │                     │
  └──m3──► B: [2,2,1] ──┘

事件分析：
  - A的m1 [1,0,0] 发生在 B接收m1 [1,1,0] 之前
    → [1,0,0] < [1,1,0] ✓

  - A发送m3时 [2,0,0]，B接收后更新为 [2,2,1]
    → B知道A的2个事件、自己的2个、C的1个

  - 若某事件VC = [2,1,1] 和另一事件VC = [1,2,1]
    → 并发！（不可比较）
    → 需要应用层冲突解决
```

## 四、向量时钟 vs 其他时钟机制

| 机制 | **空间** | **可确定因果** | **可确定全序** | **开销** | **适用场景** |
|------|---------|--------------|--------------|---------|------------|
| **物理时钟** | O(1) | 否（时钟漂移） | 否 | 低 | 粗略排序、超时 |
| **Lamport时钟** | O(1) | 是（单向） | 否 | 最低 | 简单因果 |
| **向量时钟** | O(n) | 是（双向） | 否 | 中 | 精确因果、并发检测 |
| **HLC** | O(1) | 是（近似） | 近似 | 低 | 大规模系统 |
| **TrueTime** | O(1) | 是 | 是（区间） | 高（GPS/原子钟） | Spanner |

## 五、概念定义与属性关系

| 概念 | 定义 | 属性 | 示例 | 反例 |
|------|------|------|------|------|
| **向量时钟** | 记录分布式事件因果关系的逻辑时钟 | 偏序、可检测并发、O(n)空间 | Dynamo的VClock | 物理时钟 |
| **Happens-before** | 事件e₁因果影响e₂的关系 | 传递性、非自反 | 发送→接收 | 并发事件 |
| **偏序** | 部分元素可比较的关系 | 非全序、存在不可比较元素 | 向量时钟比较 | 实数全序 |
| **并发** | 无因果关系的事件对 | 需冲突解决、不可比较 | 同时独立更新 | 因果相关事件 |
| **HLC** | 混合逻辑时钟（物理+逻辑） | O(1)空间、近似因果 | CockroachDB | 纯向量时钟 |
