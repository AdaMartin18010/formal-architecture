# 排队论基础：Little定律与M/M/1队列

> **定位**：排队论是性能工程的数学基础。Little定律（L = λW）看似简单，却是一切容量规划的基石——从线程池到K8s Pod，从数据库连接池到微服务链。
>
> **核心命题**：系统中的平均请求数 = 到达率 × 平均停留时间。这个关系在任何稳定的排队系统中成立，无论到达分布和服务分布如何。

---

## 一、思维导图：排队论核心概念

```text
排队论（Queuing Theory）
│
├─【Little定律】
│   └─ L = λW
│       L = 系统中平均请求数
│       λ = 到达率（请求/秒）
│       W = 平均停留时间（秒）
│
├─【Kendall记号】
│   └─ A/S/c
│       A = 到达过程（M=泊松，D=确定，G=一般）
│       S = 服务过程（M=指数，D=确定，G=一般）
│       c = 服务台数量
│
├─【关键模型】
│   ├─ M/M/1：单服务台，泊松到达，指数服务
│   ├─ M/M/c：多服务台（线程池模型）
│   └─ M/G/1：一般服务分布（更现实）
│
└─【工程应用】
    ├─ 线程池大小计算
    ├─ 数据库连接池
    ├─ K8s HPA目标
    └─ 微服务链延迟分析
```

## 二、Little定律的形式化与证明思路

> **权威来源**：John D.C. Little, "A Proof for the Queuing Formula: L = λW", *Operations Research*, 1961

```
Little定律：
  对于任何稳定的排队系统：
    L = λ × W

  其中：
    L = 系统中（队列+服务中）的平均请求数
    λ = 平均到达率（请求/时间单位）
    W = 请求在系统中的平均停留时间（含等待+服务）

直观理解：
  若每秒来10个请求（λ=10），每个请求平均停留0.5秒（W=0.5）
  → 系统中平均有 10 × 0.5 = 5 个请求（L=5）

关键属性：
  - 与到达分布无关（泊松、突发、均匀均可）
  - 与服务分布无关（指数、确定、重尾均可）
  - 与服务台数量无关（单台、多台均可）
  - 仅需系统"稳定"（到达率 < 服务能力）

证明思路（启发式）：
  观察时间T内：
    到达请求总数 N = λT
    每个请求贡献的"系统时间" = W
    总系统时间 = NW = λTW
    系统中平均请求数 L = 总系统时间 / T = λW
```

## 三、M/M/1队列分析

```
M/M/1模型假设：
  - 到达：泊松过程，率λ
  - 服务：指数分布，率μ
  - 单服务台
  - 无限队列容量
  - FIFO服务规则

关键公式：
  利用率（traffic intensity）：ρ = λ/μ  （需 ρ < 1）

  系统中平均请求数：L = ρ / (1 - ρ)
  队列中平均请求数：Lq = ρ² / (1 - ρ)
  平均停留时间：W = 1 / (μ - λ)
  平均等待时间：Wq = ρ / (μ - λ)

重要洞察：
  ρ → 1 时，L → ∞（拥塞崩溃）
  ρ = 0.5 → L = 1（系统轻松）
  ρ = 0.8 → L = 4（开始拥挤）
  ρ = 0.9 → L = 9（严重拥塞）
  ρ = 0.99 → L = 99（接近崩溃）

工程启示：
  目标利用率应远低于100%！
  典型目标：ρ < 0.7（留有余量应对突发）
```

## 四、工程映射

| 系统 | **λ（到达率）** | **μ（服务率）** | **排队模型** | **设计目标** |
|------|--------------|--------------|------------|------------|
| **线程池** | HTTP请求到达率 | 1/平均处理时间 | M/M/c | 线程数c使ρ≈0.7 |
| **DB连接池** | 查询到达率 | 1/平均查询时间 | M/M/c | 连接数避免等待 |
| **K8s Pod** | 请求到达率 | Pod处理能力 | M/M/c（动态c） | HPA保持ρ<目标值 |
| **微服务链** | 上游服务输出率 | 本服务处理率 | 开放Jackson网络 | 链延迟 = Σ各站W |
| **消息队列** | 消息生产速率 | 消费者消费速率 | M/M/1或M/M/c | 避免堆积（Lq） |

## 五、概念定义与属性关系

| 概念 | 定义 | 属性 | 示例 | 反例 |
|------|------|------|------|------|
| **Little定律** | L = λW，稳定排队系统的基本关系 | 普适、与分布无关、简单强大 | 计算线程池中的平均请求数 | 不稳定系统（λ > μ） |
| **利用率ρ** | 服务台忙碌时间比例（λ/μ或λ/cμ） | 关键设计参数、ρ→1时系统崩溃 | CPU利用率70% | ρ=100%（无空闲） |
| **M/M/1** | 泊松到达、指数服务、单台的排队模型 | 可解析、有闭式解 | 单线程服务器 | 多核服务器（需M/M/c） |
| **拥塞崩溃** | ρ→1时延迟和队列长度急剧恶化 | 非线性、正反馈、恢复困难 | 过载网站响应时间→∞ | 良好负载下的线性扩展 |
