# 安全模型与可信计算：安全的形式化边界

> **模块定位**：本模块是软件系统的"信任根基"层。从BAN逻辑到Rust类型系统，从认证协议到可信计算基——安全不是功能，而是**系统在所有可能攻击下的不变式**。
>
> **核心命题**：形式化安全证明的可靠性受限于理想化假设的完备性。BAN逻辑无法检测侧信道攻击，Rust类型系统无法防护硬件漏洞——证明的盲区往往比证明的内容更危险。

---

## 一、思维导图：安全形式化谱系

```text
安全模型与可信计算
│
├─【协议验证】
│   ├─ BAN逻辑（1989）→ 信念推理
│   ├─ Strand Spaces → 密码协议结构
│   └─ Tamarin Prover → 符号协议分析（自动化）
│
├─【类型系统安全】
│   ├─ Rust借用检查器 → 编译期内存安全
│   ├─ 精化类型（Refinement Types）→ 值约束
│   ├─ View类型 → 不相交字段多重可变引用
│   └─ 分离逻辑（Separation Logic）→ 堆推理
│
├─【形式化验证】
│   ├─ seL4微内核 → 函数式规约+定理证明
│   ├─ Microsoft CCF → TLA+ + 可信执行环境
│   └─ 编译器验证（CompCert）→ 证明正确编译
│
└─【局限与盲区】
    ├─ 侧信道攻击（时序、功耗、缓存）
    ├─ 硬件内存模型（TSO/ARM弱一致性）
    ├─ FFI边界（Rust Unsafe / C交互）
    └─ 规格缺陷（Selfie漏洞）
```
