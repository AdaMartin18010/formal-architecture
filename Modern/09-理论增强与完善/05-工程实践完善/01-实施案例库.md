## 1. 案例库概述

### 1.1 目标

- 收集3-5个真实项目案例
- 提供详细的实施数据（时间、成本、效果）
- 分析成功和失败因素，支撑理论验证

### 1.2 与实证研究的关系

本案例库为[02-实证研究与验证](../02-实证研究与验证/00-实证研究与验证总论.md)提供案例来源，案例数据可用于[预测准确性评估](../06-理论验证框架/02-预测准确性评估.md)。

## 2. 案例选择标准

### 2.1 基本要求

| 标准 | 要求 |
|------|------|
| **真实性** | 真实项目（非模拟、非假设） |
| **数据完整性** | 有实施时间、成本、效果数据 |
| **可追溯性** | 数据来源可验证 |

### 2.2 规模分类

| 类型 | 人月规模 | 团队规模 | 项目周期 |
|------|----------|----------|----------|
| **小型** | < 10人月 | 2-5人 | 1-3个月 |
| **中型** | 10-50人月 | 5-15人 | 3-12个月 |
| **大型** | > 50人月 | 15人以上 | 12个月+ |

### 2.3 行业覆盖

优先覆盖：电商、金融、制造、物流等典型业务系统领域。

## 4. 成功与失败因素

### 4.1 成功因素（来自理论预测）

| 因素 | 描述 | 验证指标 |
|------|------|----------|
| **技术** | 团队有编译原理背景工程师 | 关键角色到位 |
| **业务** | 核心域规则变更频率 >2次/月 | ROI充足 |
| **管理** | CTO接受DSL即代码 | Code Review纳入DSL |
| **文化** | 产品团队愿意学习DSL | 业务参与度 |
| **工具** | CI/CD支持自定义插件 | 自动化程度 |

### 4.2 失败因素（反例设计参考）

| 因素 | 描述 | 规避策略 |
|------|------|----------|
| **DSL过度复杂** | 设计过于抽象，学习曲线陡峭 | 从最小DSL起步 |
| **生成代码质量低** | 生成代码难以维护 | 沙盒测试、人工Review |
| **业务不参与** | 业务人员不参与DSL | 强制业务评审 |
| **遗留改造停滞** | 改造进度缓慢 | 渐进式、防腐层 |
| **工具链不成熟** | 工具集成困难 | 选型决策树、成熟度矩阵 |
