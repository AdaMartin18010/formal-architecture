## 1. 框架概述

### 1.1 目标

- **可追溯**：每个指标有明确的数据来源或获取方法
- **可验证**：来源可被第三方核查
- **可更新**：随实证案例补充而更新

### 1.2 来源类型

| 类型 | 说明 | 可信度 |
|------|------|--------|
| **A-实证案例** | 真实项目数据，经脱敏 | 高 |
| **B-对照实验** | 受控实验数据 | 高 |
| **C-行业基准** | 公开报告、调查 | 中 |
| **D-学术论文** | 同行评审研究 | 中 |
| **E-理论推导** | 从公理/模型推导 | 视推导严谨性 |
| **F-待收集** | 占位，待实证 | - |

## 3. 来源类型与获取方法

### 3.1 实证案例（A）

- **获取**：[实施案例库](../09-理论增强与完善/05-工程实践完善/01-实施案例库.md) 收集
- **要求**：真实项目、数据脱敏、可追溯
- **更新**：每季度补充 1-2 个

### 3.2 对照实验（B）

- **获取**：[实验设计框架](../09-理论增强与完善/02-实证研究与验证/01-实验设计框架.md) 执行
- **要求**：受控变量、统计显著性
- **更新**：按实验计划

### 3.3 行业基准（C）

- **来源**：DORA 报告、State of DevOps、Gartner、Forrester
- **获取**：公开报告、调查数据
- **注意**：需说明与语义驱动架构的映射关系

### 3.4 学术论文（D）

- **来源**：ACM、IEEE、Springer 相关会议/期刊
- **获取**：文献检索、引用
- **格式**：作者、年份、标题、会议/期刊、关键数据

## 4. 实证研究对接

### 4.1 与案例库的对接

| 案例库字段 | 对应指标 |
|------------|----------|
| 开发时间（人天） | T_c2c |
| 开发时间减少（%） | T_c2c 加速比 |
| 代码生成率（%） | R_reuse、实现最优性 |
| 缺陷率降低（%） | D_defect |

### 4.2 与实验设计的对接

详见 [01-实验设计框架](../09-理论增强与完善/02-实证研究与验证/01-实验设计框架.md) 中的因变量定义与测量方法。

### 4.3 更新流程

1. 案例/实验数据收集
2. 填入 [实施案例库](../09-理论增强与完善/05-工程实践完善/01-实施案例库.md)
3. 更新本框架「来源说明」与「状态」
4. 同步更新 [01-可度量指标定义](./01-可度量指标定义.md) 中的对比表
