# 链接修复指南

## 问题说明

文档中的"2025 对齐"章节包含大量无效链接，主要包括：

1. **示例链接**：`https://example.com/paper1`、`https://example.com/tech1` 等
2. **无效的Wikipedia链接**：使用中文标题的Wikipedia链接（如 `https://en.wikipedia.org/wiki/软件架构理论统一总论`）
3. **通用课程链接**：指向课程主页而非具体课程（如 `https://ocw.mit.edu/courses/`）

## 修复原则

### 1. Wikipedia链接修复

**错误示例**：

```markdown
- [Wikipedia: 软件架构理论统一总论](https://en.wikipedia.org/wiki/软件架构理论统一总论)
```

**正确示例**：

```markdown
- [Wikipedia: Software architecture](https://en.wikipedia.org/wiki/Software_architecture)
- [Wikipedia: Software design pattern](https://en.wikipedia.org/wiki/Software_design_pattern)
```

**规则**：

- 使用英文标题
- 使用下划线或连字符连接单词
- 确保链接指向真实存在的页面

### 2. 课程链接修复

**错误示例**：

```markdown
- [MIT: 软件架构理论统一总论](https://ocw.mit.edu/courses/)
- [Stanford: 软件架构理论统一总论](https://web.stanford.edu/class/)
```

**正确示例**：

```markdown
- [MIT 6.033: Computer Systems Engineering](https://web.mit.edu/6.033/www/)（软件架构与系统设计）
- [CMU 17-313: Foundations of Software Engineering](https://www.cs.cmu.edu/~ckaestne/17313/)（软件架构实践）
- [Stanford CS 244: Advanced Computer Systems](https://web.stanford.edu/class/cs244/)（分布式系统架构）
```

**规则**：

- 指向具体课程页面
- 包含课程编号和名称
- 添加简短说明

### 3. 论文链接修复

**错误示例**：

```markdown
- [Recent Paper 1](https://example.com/paper1)
- [Recent Paper 2](https://example.com/paper2)
```

**正确示例**：

```markdown
- [Architectural Styles and the Design of Network-based Software Architectures](https://www.ics.uci.edu/~fielding/pubs/dissertation/top.htm) (Fielding, 2000)
- [Software Architecture: Perspectives on an Emerging Discipline](https://www.cs.cmu.edu/~able/publications/softwareArchitectureBook.pdf) (Shaw & Garlan, 1996)
```

**规则**：

- 使用真实论文标题
- 链接到可访问的PDF或论文页面
- 包含作者和年份信息

### 4. 技术链接修复

**错误示例**：

```markdown
- [Technology 1](https://example.com/tech1)
- [Technology 2](https://example.com/tech2)
```

**正确示例**：

```markdown
- [Kubernetes](https://kubernetes.io/)（容器编排平台）
- [Istio](https://istio.io/)（服务网格）
- [OpenTelemetry](https://opentelemetry.io/)（可观测性标准）
```

**规则**：

- 使用真实技术名称
- 链接到官方网站
- 添加简短说明
