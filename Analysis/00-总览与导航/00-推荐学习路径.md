# 推荐学习路径

[返回主题树](00-主题树与内容索引.md) | [主计划](00-形式化架构理论统一计划.md)

> 按**先修关系**与认知层级规划的学习路径，便于按「快速掌握」顺序阅读。每条路径注明对应本仓库文档及可选的课程/标准参照。
>
> **路径选择建议**：无形式化或数学基础者建议走**路径一**（主轨）；已有编程与类型论/工程背景者可走**路径二**或**路径四**（极短轨仅软件架构）。

**为何推荐间隔重复与检索练习**：认知心理学与学习科学表明，**间隔重复**（将学习分散到多天、多周）和**检索练习**（闭卷回忆后再核对）比集中复习、被动重读更能促进长期记忆与迁移；学习者常误以为「集中复习、反复阅读」更有效，是因为当下流畅感更强，但长期保留往往不如间隔+检索。本路径中的「复习节点」「自检问题」「先自答再对照」即据此设计；详见综述如 *Nature Reviews Psychology* 等对 spacing 与 retrieval practice 的元分析。

## 路径一：形式化基础 → 形式模型 → 架构（主轨）

### 可选：先修自测（建议路径一学习者）

在进入形式模型与验证前，建议先确认以下基础（可自测 3–5 题，答对再进 [06-形式模型理论体系总论](../06-形式模型理论体系/00-形式模型理论体系总论.md)，避免过早高负荷）：

- **集合与关系**：集合运算、关系、函数；见 [02-数学理论体系总论](../02-数学理论体系/00-数学理论体系总论.md) 前几节。
- **命题逻辑与一阶逻辑**：命题联结词、量词、基本推理；见 [01-哲学基础理论总论](../01-哲学基础理论/00-哲学基础理论总论.md) 逻辑学部分。
- **自动机/形式语法**（可选）：有限状态机、形式语法四元组；见 [03-形式语言理论体系总论](../03-形式语言理论体系/00-形式语言理论统一总论.md) 概述。

自测方式：闭卷写出上述 2–3 个概念的定义或举例，再对照上述文档核对。

1. **哲学与数学基础**
   [01-哲学基础理论总论](../01-哲学基础理论/00-哲学基础理论总论.md) → [02-数学理论体系总论](../02-数学理论体系/00-数学理论体系总论.md)
   - 可选参照：SEBoK Part 2–3、本科离散数学/逻辑课程。

2. **形式语言与自动机**

   [11-理论统一与整合/07-理论统一与整合总论](../11-理论统一与整合/07-理论统一与整合/00-理论统一与整合总论.md)。

### 复习节点（间隔与检索练习）

建议在以下节点进行**回顾**，以强化长期记忆（间隔重复）。**建议复习间隔**：学完新模块后 1–2 周内回顾前置模块，可结合检索练习自检问题。

- **学完 03-形式语言后**（建议 1–2 周内）：回顾 [02-数学理论体系](../02-数学理论体系/00-数学理论体系总论.md) 中与语法/语言相关的集合与关系。
- **学完 06-形式模型后**：回顾 [03-形式语言](../03-形式语言理论体系/00-形式语言理论统一总论.md) 中自动机、形式语法与形式模型的关系。
- **学完 06-软件架构后**：回顾 [06-形式模型](../06-形式模型理论体系/00-形式模型理论体系总论.md) 中状态机、Petri 网与架构模式的关系。
- **学完 07-分布式后**：回顾 [06-软件架构](../06-软件架构理论体系/00-软件架构理论体系总论-整合版.md) 与形式模型中的并发/时序。

### 自检问题与回顾概念列表

每模块建议自检 2–3 个问题，并与 [00-核心概念表（形式化架构）](00-核心概念表（形式化架构）.md) 联动：

| 模块 | 自检问题 / 回顾概念 |
|------|---------------------|
| 01-哲学 | 一阶逻辑公理、ZFC 外延公理、存在与数学对象的本体论 |
| 02-数学 | 集合运算、范畴的态射与函子、类型与项 |
| 03-形式语言 | 形式语法四元组、乔姆斯基层次、指称/操作/公理语义 |
| 06-形式模型 | FSM、Petri 网、LTL/CTL、模型检查、进程代数（CCS/CSP） |
| 05-编程语言 | 类型系统、指称/操作语义、编译与解释 |
| 06-软件架构 | 架构模式（分层/MVC/微服务/事件驱动）、组件与连接器、ISO 42010 视点 |
| 07-分布式 | 分布式系统四元组、一致性、CAP、微服务通信 |

**检索练习引导**：建议先**闭卷自答**上表与各总论「本节要点与自检」中的问题，再对照 [00-核心概念表（形式化架构）](00-核心概念表（形式化架构）.md) 或正文核对，以强化提取练习（retrieval practice）与长期记忆。Modern 路径的复习与自检与本文档一致，见 [Modern/00-学习路径推荐](../../Modern/00-学习路径推荐.md)。

**工作样例（归纳与迁移）**：对需要归纳与迁移的子域，可结合具体文档做「阅读—归纳—应用」：形式化验证见 [08-实践应用开发/03-自动化验证工具设计与实现](../08-实践应用开发/03-自动化验证工具设计与实现.md)；架构评估与模式应用见 [07-分布式/04-软件架构/01a-设计模式详解](../07-分布式与微服务/04-软件架构理论体系/01a-设计模式详解.md)、[10-统一架构质量系统理论](../07-分布式与微服务/04-软件架构理论体系/10-统一架构质量系统理论.md)。

**归纳与迁移检查清单**（学完工作样例后建议自答）：

- 该样例体现了哪些形式模型/架构/语义概念？（归纳：提取通用模式）
- 如何将样例中的概念迁移到我的项目或问题域？（迁移：应用抽象）
- 本样例与 XX 理论/文档的关系是什么？（关联：建立知识连接）
- 我能用一句话向他人解释本样例的核心思想吗？（精炼：检验理解）

**各体系工作样例入口**（每体系 1–2 个含最小示例或与标准对应的文档）：01-哲学 → 总论中逻辑学与本体论小节；02-数学 → [08-类型论基础](../02-数学理论体系/08-类型论基础.md)；03-形式语言 → 总论+[04-语义理论对比](../03-形式语言理论体系/04-语义理论对比.md)；06-形式模型 → [04-模型检查理论](../06-形式模型理论体系/04-模型检查理论.md)；05-编程语言 → [00-类型理论-范畴论基础](../05-编程语言理论体系/00-类型理论-范畴论基础-2025版.md)；06-软件架构 → [01a-设计模式详解](../07-分布式与微服务/04-软件架构理论体系/01a-设计模式详解.md)、[10-统一架构质量系统理论](../07-分布式与微服务/04-软件架构理论体系/10-统一架构质量系统理论.md)；07-分布式 → 07-分布式总论 + 04-软件架构子体系。

### 跨模块对比阅读（交织学习）

为区分易混淆概念，建议在以下顺序做**跨模块对比**：

- [11-理论统一与整合总论](../11-理论统一与整合/07-理论统一与整合/00-理论统一与整合总论.md)：统一状态转换（状态机 vs Petri 网）、统一模块化（组件 vs 接口）。
- [07-分布式/04-软件架构理论体系](../07-分布式与微服务/04-软件架构理论体系/README.md) 与 [06-软件架构理论体系](../06-软件架构理论体系/)：分布式视角下的架构 vs 主体系架构。

**05-编程语言 ↔ 03-形式语言 对比阅读**：学完 [05-编程语言总论](../05-编程语言理论体系/00-编程语言理论统一总论.md) 第 3 节（语义理论）后，建议穿插阅读 [03-形式语言总论](../03-形式语言理论体系/00-形式语言理论统一总论.md) §1.3 形式语义、§4 形式语义理论，以及 [04-语义理论对比](../03-形式语言理论体系/04-语义理论对比.md)，再回 05 总论 §4（类型系统）。可区分「编程语言语义」与「形式语言语义」的对应关系与层次。

**06-形式模型 ↔ 03-形式语言 对比阅读**：学完 [06-形式模型总论](../06-形式模型理论体系/00-形式模型理论体系总论.md) §3（Petri 网）后，建议穿插阅读 [03-形式语言总论](../03-形式语言理论体系/00-形式语言理论统一总论.md) §2 乔姆斯基层次、§1.2 形式语法（自动机与形式语法的关系），再回 06-形式模型 §4 时序逻辑、§6 自动机理论。可强化「形式语法/自动机」与「形式模型（状态机/Petri/时序）」的迁移。

**建议交织顺序**（穿插阅读以强化迁移）：

- 学完 [06-形式模型总论](../06-形式模型理论体系/00-形式模型理论体系总论.md) §3（Petri 网）后 → 穿插阅读 [03-形式语言总论](../03-形式语言理论体系/00-形式语言理论统一总论.md) §2（形式语法/自动机）→ 再回 06-形式模型 §4（时序逻辑）。
- 学完 [05-编程语言总论](../05-编程语言理论体系/00-编程语言理论统一总论.md) §3（语义理论）后 → 穿插阅读 [03-形式语言/04-语义理论对比](../03-形式语言理论体系/04-语义理论对比.md) → 再回 05 §4（类型系统）。
