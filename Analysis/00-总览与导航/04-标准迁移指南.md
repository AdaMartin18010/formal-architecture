## 1. 迁移总览

### 1.1 迁移路径图

```
┌─────────────────────────────────────────────────────────────────┐
│ 来源                    迁移目标                                 │
├─────────────────────────────────────────────────────────────────┤
│ IEEE 42010:2011        → IEEE 42010:2022（架构描述）             │
│ ISO 42020:2019         → 保持，补充 42024/42042 对齐             │
│ RDF 1.1 / OWL 2        → RDF 1.2 / OWL 2（SHACL 1.2 对齐）       │
│ SPARQL 1.1             → SPARQL 1.2（开发中）                    │
│ 无 SHACL               → SHACL 1.2 Core/SPARQL/Rules              │
│ MDA / DDD / 语义网      → SMDD（语义模型驱动设计）               │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 迁移优先级

| 优先级 | 迁移项 | 影响范围 | 建议时间 |
|--------|--------|----------|----------|
| P0 | 42010:2011 → 42010:2022 | 架构描述文档 | 立即 |
| P0 | SHACL 1.2 采纳 | 语义模型验证 | 2025 Q2 |
| P1 | 42024/42042 预对齐 | 架构基础、参考架构 | 标准发布后 |
| P1 | RDF 1.2 / SPARQL 1.2 | 知识图谱、语义网 | 标准发布后 |
| P2 | MDA/DDD 到 SMDD | 业务建模、架构设计 | 按项目需求 |

## 2. ISO 42000 架构标准族迁移

### 2.1 IEEE 42010:2011 → 42010:2022

| 变更点 | 2011 | 2022 | 迁移行动 |
|--------|------|------|----------|
| 架构描述 | 视图、视点 | 保持，术语微调 | 检查视图定义一致性 |
| 系统 | 软件、系统 | 扩展企业、SoS | 补充企业架构视图 |
| 利益相关方 | 基础 | 增强 | 更新利益相关方分析 |

**迁移步骤**：

1. 审查现有架构描述文档
2. 对照 42010:2022 更新视图-视点定义
3. 补充缺失的利益相关方关注点

### 2.2 42024（架构基础）预采纳

- **状态**：DIS，2025 征求意见
- **行动**：术语预对齐，见 [02-ISO42024架构基础标准对齐](../06-软件架构理论体系/02-ISO42024架构基础标准对齐.md)
- **正式采纳**：标准发布为 IS 后

### 2.3 42042（参考架构）预采纳

- **状态**：DIS，2025 征求意见
- **行动**：参考架构要求预对齐，见 [03-ISO42042参考架构标准对齐](../06-软件架构理论体系/03-ISO42042参考架构标准对齐.md)
- **领域**：AI/ML、云计算、智能制造等

## 3. 语义网标准迁移（RDF/OWL/SPARQL/SHACL）

### 3.1 RDF 1.1 → RDF 1.2

| 变更点 | 1.1 | 1.2 | 迁移行动 |
|--------|-----|-----|----------|
| 语义 | 2014 推荐 | 2026 工作草案 | 关注 RDF 1.2 语义更新 |
| 与 SHACL 1.2 | 独立 | 对齐 | 联合迁移 |

**迁移步骤**：

1. 跟踪 W3C RDF 1.2 工作草案
2. 标准发布后更新 RDF 处理工具
3. 验证与 SHACL 1.2 的兼容性

### 3.2 无 SHACL → SHACL 1.2

| 能力 | 无 SHACL | SHACL 1.2 | 迁移行动 |
|------|----------|-----------|----------|
| 图约束 | 无/自定义 | Core 形状约束 | 定义 SHACL 形状 |
| SPARQL 扩展 | 无 | shacl:sparql | 复杂约束用 SPARQL |
| 规则 | 无 | Datalog 风格规则 | 派生属性、推理规则 |

**迁移步骤**：

1. 在 `Modern/09-理论增强与完善/01-形式化证明增强/` 中新增 SHACL 验证方法
2. 为 MSMFIT/RDF 映射定义 SHACL 形状
3. 建立语义模型验证流水线

### 3.3 SPARQL 1.1 → 1.2

- **状态**：SPARQL 1.2 开发中
- **行动**：标准发布后更新查询、与 SHACL 1.2 联合验证

详见 [Modern/09-理论增强与完善/03-国际对标深化/00-综合迁移与融合指南](../../Modern/09-理论增强与完善/03-国际对标深化/00-综合迁移与融合指南.md)。

**快速索引**：

- **MDA → SMDD**：UML 模型 → MSMFIT → DSL → 代码
- **DDD → SMDD**：统一语言 → MSMFIT → 限界上下文 → 聚合根
